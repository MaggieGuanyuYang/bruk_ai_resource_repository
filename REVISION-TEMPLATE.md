# Monthly Revision Request

Fill in this template each month to request content updates. Reference exact
pages, sections, and source files using the names from
[SITE-STRUCTURE.md](SITE-STRUCTURE.md).

> **Note:** Every change is a YAML edit. The published `*.html` files are
> generated by `python3 build.py` and committed alongside the YAML. Once the
> revision is applied, both the source and the regenerated HTML get committed
> in the same commit so the live GitHub Pages site updates immediately.

---

## Revision metadata

- **Revision date:** `YYYY-MM-DD`
- **Requested by:** _Name / role_
- **Reason for update (optional):** _e.g. new tool launched, broken link reported, periodic refresh_

## Home-page-only edits — `content/home.yml`

> The home page has its own shape (hero band + section grid + audience
> tiles + "What this is" prose) and isn't covered by the generic per-page
> tables below. Use these rows for any home-page change.

| Action                                          | YAML path                                               | Details                                                       |
| ----------------------------------------------- | ------------------------------------------------------- | ------------------------------------------------------------- |
| **Update the hero eyebrow** (small line above H1) | `hero.eyebrow`                                        |                                                               |
| **Update the hero H1**                          | `hero.heading`                                          |                                                               |
| **Update the hero lede**                        | `hero.lede`                                             | Markdown allowed.                                             |
| **Change a hero button label**                  | `hero.actions[<n>].label`                               | `n=0` is the primary teal button, `n=1` the ghost button.     |
| **Re-point a hero button**                      | `hero.actions[<n>].ref` or `.href`                      | Use `ref:` for an on-site page (page ID); `href:` for an external URL or `#anchor`. |
| **Edit a section card's blurb**                 | `section_cards[<n>].desc`                               | `n=0…5` in display order: Section 1, Section 2, Section 3, Section 4, About, Section 5. |
| **Edit a section card's title**                 | `section_cards[<n>].title`                              |                                                               |
| **Edit a section card's "meta" footer line**    | `section_cards[<n>].meta`                               | e.g. "5 sub-pages", "Coming 2026"                             |
| **Update an audience tile**                     | `audience[<n>].heading` / `.body`                       | `n=0` "New to AI", `n=1` "Short on time", `n=2` "Planning a grant or paper" |
| **Replace an audience tile's icon**             | `audience[<n>].icon`                                    | Inline SVG string. Match the existing 18×18 outline style.    |
| **Edit a "What this is — and isn't" paragraph** | `what.paragraphs[<n>]`                                  | Three paragraphs by default; HTML inline tags allowed (`<a>`, `<strong>`, `<em>`). |
| **Override the home-page footer text**          | top-level `footer_about` / `footer_contact`             | Home overrides `_site.yml` defaults (e.g. it lists the Privacy Statement PDF). |

## Per-page content changes

Use one block per page that needs editing. Page IDs and their YAML sources
come from the [SITE-STRUCTURE.md page index](SITE-STRUCTURE.md#page-index).

### Page

- **Page ID:** _e.g._ `section-1.literature-review`
- **Source file:** _e.g._ `content/section-1/literature-review.yml`

#### Tool grid edits

> Pages with `tool_grid:` blocks: `section-1.literature-review`,
> `section-1.hypothesis-generation`, `section-1.data-analysis`,
> `section-1.writing-reporting`, `section-3` (online courses, YouTube,
> advanced learning), and a couple of inline grids in section-2 sub-pages.

| Action                              | Where to edit (YAML path)                          | Details                                           |
| ----------------------------------- | -------------------------------------------------- | ------------------------------------------------- |
| **Add a tool**                      | append a `{ … }` to the `tool_grid:` list           | `name`, `url` (optional), `tag` (optional), `desc`, `link_url`/`link_text` (optional) |
| **Update a tool's URL**             | `tool_grid[<n>].url`                               | New URL                                           |
| **Update a tool's description**     | `tool_grid[<n>].desc`                              | New blurb                                         |
| **Add a "Free / paid" tag**         | `tool_grid[<n>].tag`                               | e.g. `Free`, `Free &amp; paid`                    |
| **Add a video walkthrough link**    | `tool_grid[<n>].link_url` + `link_text`             | URL + button text                                 |
| **Remove a tool**                   | delete the corresponding `{ … }` entry             |                                                   |

#### Prose / paragraph edits

| Action                              | Where to edit                                      | Details |
| ----------------------------------- | -------------------------------------------------- | ------- |
| **Tweak the lede paragraph**        | `body[…].prose[0].lede`                            | Markdown allowed inside the lede string.          |
| **Edit a section's prose**          | `body[…].prose[<n>].markdown`                      | The block under whichever `h2`/`h3` you mean. Use markdown; references to other pages can stay as plain links. |
| **Fix a broken link**               | search for the URL in the YAML; replace            | The build will not warn — verify the new URL works. |
| **Add a citation/reference**        | append to the relevant `references.items[]`        | One item per citation, markdown allowed for italic and DOI links. |
| **Add a callout**                   | insert a new `- callout: { heading: "…", body: "…" }` block | Optional `kind: caution` or `kind: warning` for the orange/red style. |

#### Headings / on-this-page nav

| Action                              | Where to edit                                      | Details |
| ----------------------------------- | -------------------------------------------------- | ------- |
| **Rename an `h2`**                  | the `h2.text` field of that block                  | If you change the `id`, update `quicknav.items[]` to keep the anchor working. |
| **Add an "On this page" entry**     | `body[…].prose[…].quicknav.items[]`                | `{ href: "#anchor", label: "…" }`                 |

#### Page metadata

| Action                              | Where to edit                                      | Details |
| ----------------------------------- | -------------------------------------------------- | ------- |
| **Update the SEO description**      | top-level `description:`                           | One sentence, max ~155 chars.                     |
| **Change the breadcrumb trail**     | `breadcrumb[]`                                     | List of `{ ref, label }` entries; the last entry should be the current page (no `ref`). |
| **Update the badge above the H1**   | top-level `badge:`                                 | e.g. "Section 2 · Topic 4"                        |

#### Pager (prev/next at the bottom)

| Action                              | Where to edit                                      | Details |
| ----------------------------------- | -------------------------------------------------- | ------- |
| **Re-point next/prev**              | `pager.prev` / `pager.next`                        | `{ ref: "<page-id>", label: "…", title: "…" }`    |
